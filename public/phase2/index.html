An Internet of Things Project
by Matt Lindly and Sam Lees

Part of "Introduction to Internet of Things (CPE 458)" tought by David Maluf, Raghuram Sudhaakar, and Foaad Khosmood, a partnership between Cisco Systems, Inc. and California Polytechnic State University, San Luis Obispo in Fall 2016.

Note:  Significant structural modifications, performance improvements, and additional features are in progress for future releases as part of this project's continuation as a senior project. The work to be submitted for Senior Project credit will be both a significant enhancement of the original code and a functionally superior product with substantial additional features.

Background

The problem that the SmartGarden project seeks to solve is the challenge of watering succulents correctly and maintaining consistent plant health. Too much water can cause root rot and drowning while too little water can cause dehydration and starvation.
The SmartGarden solves this common problem by configuring a Raspberry Pi to measure soil moisture, store the data to the cloud, and inform the user appropriately by providing a web-interface and a variety of notifications.

Technical Details

Some challenges of this project included handling the analog-to-digital conversion. This was overcome by purchasing a MCP3008 ADC to convert the output of the sensor and provide the Pi with compatible input. Implementing CoAP was also a challenge, especially with regard to handling packets that arrived out of order. Another challenged that arose due to the end-to-end nature of our project was cross-language integration. Careful calls of fork() and exec() had to be introduced to the CoAP code to run various sensor and notification processes. Designing a database schema also proved to be a challenge. The final solution utilizes a SQLite database for all data storage. Finally, ensuring data integrity and performing analytics on the data collected created the need for a few algorithms, such as a simple rolling average to reduce sensor noise.

Technical Implementation

The project naturally lent itself to a three-tiered approach. The first iteration of the project involved developing a solution to run locally on the node and provide sensor readings. Once this step was complete, the functioning node was integrated with the previously developed CoAP communication protocol. After the client was successfully communicating with the server, a database was implemented to store the data and a variety of analytics was performed to provide an accurate user-interface and notification platform.

Demonstration


A successful demonstration of the product showcases the web-interface with real-time moisture gauges and graphs for historical data at various intervals. The IFTTT integration for notifications is also demonstrated showing that push notifications, emails, and phone calls are actuated when action is needed or performed on the system.

Conclusion

A few features that could be implemented in the future to enhance this product include predictive notifications. This feature would allow users to receive notifications in advance of when their succulent needs water so that they could attend to the task at their convenience without risking plant health. Additional features include automated watering either by integrating with an existing smart watering system or by creating another system to pair with the SmartGarden. The system also can be scaled with up to eight sensors per node. The scaling of the existing product to multiple nodes would require implementation of authentication protocols as well as encryption to secure each user’s personal data.
